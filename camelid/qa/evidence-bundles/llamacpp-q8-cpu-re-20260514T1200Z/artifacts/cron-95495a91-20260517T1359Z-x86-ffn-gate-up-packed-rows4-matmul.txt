Design:
- New gate: CAMELID_X86_Q8_FFN_GATE_UP_PACKED_ROWS4_MATMUL=on.
- Fallback stays closed unless the resolved runtime plan enables the gate, input is rank-2 with more than one row, input width is Q8_0 block aligned, both blk.*.ffn_gate.weight and blk.*.ffn_up.weight are Q8_0RuntimeStorage::PackedRows4, both are I8 interleave, packed block geometry matches, and output widths match.
- The path quantizes the shared multi-row activation input once with q8_0_quantized_matmul_input_rows, then reuses those blocks for both gate and up projections via q8_0_packed_rows4_matmul_projection_from_quantized.
- No duplicate packed-copy sidecar was added; selected tensors remain backend-owned runtime packed storage and existing safe paths handle fallback.
