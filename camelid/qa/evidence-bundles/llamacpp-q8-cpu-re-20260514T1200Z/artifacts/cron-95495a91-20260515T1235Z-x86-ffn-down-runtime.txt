# UBUNTU_X86_Q8 follow-on slice: FFN down runtime-owned packed storage

Scope:
- Ubuntu x86_64 dense Llama Q8_0 only.
- Small technical slice: widen default-off `CAMELID_X86_Q8_REPACK=on` runtime-packed Q8_0 load gate to include `blk.*.ffn_down.weight` alongside the existing dense attention projection family and gate/up FFN rows.
- Fallback: unchanged when `CAMELID_X86_Q8_REPACK` is unset/off, when `CAMELID_Q8_0_BLOCK_DOT=off`, when shape is not rows4/block aligned, or when a tensor is outside the selected dense Llama family.
- Claim boundary: loader/runtime-storage correctness only; no support-contract, production-throughput, portability, accelerator, VNNI/AVX512, or non-Llama-family claim.

llama.cpp-informed notes:
- The lane still tracks llama.cpp's CPU `GGML_OP_MUL_MAT` Q8_0 x Q8_0 path, where x86 evidence points at AVX2-era tinyBLAS/llamafile code rather than VNNI/AVX512 as selected hot evidence.
- `blk.*.ffn_down.weight` is the large dense Llama FFN projection that maps FFN width back to hidden width. Camelid stores the GGUF descriptor shape, but the runtime hot path consumes transposed rows. This slice makes the backend-owned packed runtime storage directly match that transposed consumption shape instead of keeping a duplicate row-major packed sidecar.

Implementation evidence:
- `src/tensor/mod.rs`:
  - `q8_repack_x86_tensor_enabled` now includes `*.ffn_down.weight`.
  - `q8_repack_linear_shape` treats gate/up/down FFN descriptors as transposed runtime rows `[output, input]`.
  - Existing runtime storage remains `Q8_0RuntimeStorage::PackedRows4`; no q8_0 block copy, file backing, or old packed sidecar is retained when this path succeeds.
- `tests/tensor_store.rs`:
  - `x86_q8_repack_loads_dense_ffn_family_as_transposed_packed_runtime` covers gate/up/down.
  - The down case uses descriptor dims `[64,32]` and verifies runtime packed rows `[32,64]` (`rows=32`, `blocks_per_row=2`).
