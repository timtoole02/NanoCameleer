# CAMELID BACKEND ENGINEER UBUNTU X86 Q8 — managed decode-consumer flags

Timestamp: 2026-05-17T00:35Z
Scope: Ubuntu x86_64 dense Llama Q8_0 only.
Canonical host check: `SSH to the canonical validation host` reported Ubuntu 24.04, x86_64, 16 CPUs, Intel Xeon Platinum 8488C, with AVX2/AVX512/VNNI/AMX CPU flags available.

## Small technical slice

ExecutionPlan now manages the default-off x86 Q8 decode-consumer flags for attention QKV, attention projection, and FFN gate/up in addition to the existing x86 repack/kernel/output/FFN-down flags. This prevents stale opt-in owner experiments from leaking into planned runs and keeps the safe fallback behavior explicit.

This is a planner/runtime-gate hardening slice, not a throughput promotion. It does not add a duplicate packed-copy sidecar. The directly usable paths continue to consume backend-owned `Q8_0RuntimeStorage::PackedRows4` and remain default-off.

## Fallback / claim boundary

- With `CAMELID_PROFILE` unset/auto or without explicit `CAMELID_X86_Q8_REPACK=on CAMELID_X86_Q8_KERNEL=avx2`, Ubuntu x86 Q8 planning remains `cpu_reference` / `safe_q8_0_block_dot`.
- With current validated experimental gates, ExecutionPlan still selects the runtime-packed AVX2 path but now pins attention QKV/projection, FFN gate/up, FFN-down, and output owner/consumer flags to `off` unless separately opted into a developer run.
- No performance, support-contract, portability, or broader model-family claim is made by this slice.
